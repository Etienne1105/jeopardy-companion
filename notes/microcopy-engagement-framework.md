# Référentiel microcopie & engagement — Jeopardy! Companion

> Source : recherche fournie par Etienne (2026-06-15). ~40 règles, 6 catégories. À appliquer à TOUS les textes/feedback de l'app.

## ⭐ Règle maîtresse + ordre de priorité (en cas de conflit)
**Clarté (A2) > Action claire (B1) > Compétence préservée (C7/F2) > Personnalité/célébration (Peak-End D3).**
Toujours écrire la version CLAIRE d'abord, puis superposer la personnalité SEULEMENT si elle ne coûte aucune clarté. Jamais d'humour forcé.

## A — Voix & ton
- **A1** Voix constante, ton variable selon l'état émotionnel (succès=enthousiaste, erreur=empathique sobre, chargement=léger).
- **A2** Clarté > personnalité. Les gens lisent ~20-28% des mots.
- **A3** Voix active, présent, langage positif, contractions, adresse directe, pas de jargon ni double négation.
- **A4** Identifier la fonction avant d'écrire : Informer / Influencer / Interagir.

## B — UX writing par composant
- **B1 CTA** : verbe + objet + bénéfice, ~2 mots, sentence case. Bannir « Soumettre / Cliquez ici / OK ». 1re personne (« Voir mon score ») > possessif. Décrire ce qui se passe après le clic.
- **B2 Erreurs** : [ce qui s'est passé] + [comment corriger]. Jamais de blâme ni de code brut. Pas d'humour sur erreur bloquante.
- **B3 États vides** : titre + motivation + 1 CTA. Jamais « Aucun résultat » seul. (Ex. « Tu as fini tous les quiz du jour 🎉 ».)
- **B4 Placeholders ≠ labels** : label persistant descriptif ; placeholder = exemple de format.
- **B5 Succès** : confirmer en positif + bref + étape suivante.
- **B6 Chargement** : feedback immédiat si > 400 ms (Doherty). La microcopie de chargement peut porter la personnalité.

## C — Boucles d'engagement & gamification
- **C1 Hook (Eyal)** : Trigger → Action facile → Récompense variable → Investissement. Chaque session = un hook.
- **C2 Fogg B=MAP** : réduire la friction AVANT d'augmenter la motivation (« Une question, 10 s »).
- **C3 Récompenses variables** ⚠️ : varier les félicitations / micro-récompenses. Garde-fou : pas de piège/addiction.
- **C4 Goal-Gradient** : toujours afficher la progression, accentuer la fin (« Plus que 2 ! »).
- **C5 Endowed Progress** : **barre jamais à zéro**. Créditer la 1re étape (intro lue, catégorie choisie). « Tu as déjà commencé ! »
- **C6 Streaks** ⚠️ : compteur visible + **mécanisme de pardon obligatoire** (streak freeze). Garde-fou : anxiété / streak creep → offrir des sorties.
- **C7 SDT (Autonomie/Compétence/Appartenance)** : choix (catégorie/difficulté) ; feedback valorisant le progrès ; langage inclusif. Badges mal calibrés = sur-justification (nuit à la motivation).
- **C8 Octalysis** : checklist ; privilégier White Hat (sens, accomplissement, créativité) > Black Hat (rareté, peur de perte).
- **C9 Juiciness (Kao 2020, U inversé)** : bonne réponse = animation + son + microcopie. **Dosage moyen-élevé** ; ni absence, ni excès.

## D — Lois UX de session (Yablonski)
- **D1 Zeigarnik** : montrer l'incomplétude (« 3 restantes »), cliffhangers entre sections.
- **D3 Peak-End (Kahneman)** : soigner le **pic** (question bonus, micro-célébration) et la **fin** (écran de résultat mémorable). C'est là qu'on met la personnalité.
- **D4 Doherty** : feedback < 400 ms.
- **D5 Hick** : 3-4 réponses max, un seul CTA primaire.
- **D6 Aesthetic-Usability** : beau = perçu plus fiable. Soigner la finition des écrans de feedback.
- **D7 Von Restorff** : un seul élément saillant par écran (CTA primaire, bonne réponse révélée).
- **D8 Miller/Chunking** : découper en sections nommées.

## E — Psychologie cognitive du quiz
- **E1 Curiosity Gap (Loewenstein)** : ouvrir l'écart (« Tu penses connaître ? ») puis **garantir la résolution**. Gap trop large = frustration.
- **E2 Testing Effect (ÉTABLI)** : inviter à prédire avant les options (« Devine d'abord ») ; après réponse, **toujours** la bonne réponse + une ligne de « pourquoi ».
- **E3 Rétroaction immédiate (ÉTABLI)** : feedback dès la réponse (par défaut immédiat).
- **E4 Résultats personnalisés (Barnum)** : identité valorisante (« Tu es un Stratège »), libellé court partageable. ⚠️ Engagement, pas vérité psychométrique.

## F — Microcopie de feedback (quiz)
- **F1 Bonne réponse** : confirmation positive **variée** (« Exact ! », « Bien vu », « Tu gères ») + micro-explication + juice.
- **F2 Mauvaise réponse (LEVIER CRITIQUE)** : [marqueur doux] + [bonne réponse] + [pourquoi] + [encouragement]. Jamais de rouge agressif/blâme/« WRONG ». « Make failure feel safe » (Duolingo). Reformuler l'erreur en info.
- **F3 Score** : contextualiser positif (« 7/10 — solide ! »), progrès vs soi-même, scores faibles = marge de progression.
- **F4 Progression** : barre jamais à zéro + proximité (« Plus que 2 ! ») + tension (« 3 restantes »).
- **F5 Fin (Peak-End)** : célébration max + résultat valorisant + **boucle ouverte** vers la suite (« Prêt pour le niveau 2 ? »).
- **F6 Relance** : déclencheur avec bénéfice clair, jamais culpabilisant.

## Application à CETTE app (Géopardie/Jeopardy!)
- **Bannière champion (streak)** = Peak (D3) + identité (E4) + Johnny Gilbert (appartenance/célébration).
- **Reveal résultat réel** = refermer le curiosity gap (E1).
- **Médailles ! / couronne** = compétence (C7) + juice (C9, dosage moyen-élevé) ; couronne = pic rare.
- **Reset/score** = F3, jamais humiliant.
- Quiz **éducatif** (Jeopardy = trivia) → maximiser testing effect (E2) + feedback élaboré.
- Rétention → streaks + SDT AVEC garde-fous + streak freeze.

## Caveats
Établis : testing effect, rétroaction immédiate, goal-gradient, endowed progress, Doherty, Zeigarnik, Peak-End, curiosity gap, SDT. À nuancer : streaks/récompenses variables (dark patterns, streak creep), Octalysis (cadre praticien), juiciness (U inversé). Stats CTA/Duolingo = directionnelles (A/B propriétaires / blog interne). Effet Barnum = engagement, pas mesure.
